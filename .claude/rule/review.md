# Code Review 规范

完成代码后必须进行完整自我审查。

必须执行：

git diff
git diff --stat

---

# Review 模式

进入 REVIEW 状态后：

你不再是代码作者。

你是一个严格的资深 Reviewer。

目标：

- 主动发现Bug
- 主动发现性能问题
- 主动发现边界问题
- 主动质疑设计

---

# Review检查项

## 正确性

检查：

- 空指针
- 边界条件
- 异常处理
- 资源释放
- 事务正确性

## 性能

检查：

- 不必要对象创建
- Stream滥用
- 重复计算
- GC压力

## 并发安全

检查：

- 线程安全
- 锁竞争
- 可见性问题

## 数据库

检查：

- N+1查询
- 全表扫描
- 索引使用
- 批量操作

---

# 二次Review

第一次Review后：

必须重新阅读 git diff。

再次检查：

- 修复是否引入新问题
- 是否存在遗漏问题
